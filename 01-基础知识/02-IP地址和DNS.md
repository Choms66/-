## 📖 第1部分：IP地址 - 网络上的地址

### 想象一个例子：寄信

在现实生活中，如果你要寄信给朋友：

```
你的信
┌─────────────────┐
│ 收信人：小红      │
│ 地址：北京市朝阳区│
│     某某街123号   │
│ 邮编：100000      │
└─────────────────┘
   ↓
邮递员用地址找到小红的家
   ↓
送信！
```

**在网络里，IP地址就是这个"地址"！**

### 什么是IP地址？

**IP地址** = 计算机的网络地址

就像每个人的家都有地址（邮编、街道、门号），每台电脑在网络上也有一个"地址"，叫IP地址。

### IP地址长什么样？

**IPv4地址**（现在用的最多）：
```
192.168.1.1
^   ^    ^ ^
│   │    │ └─ 第4段（0-255）
│   │    └──── 第3段（0-255）
│   └───────── 第2段（0-255）
└───────────── 第1段（0-255）
```

**4段数字，用点分开，每段0-255**

### 例子：

| IP地址 | 代表什么 |
|-------|--------|
| **192.168.1.1** | 你家里的路由器 |
| **192.168.1.100** | 你电脑的地址（家庭网络里） |
| **8.8.8.8** | Google的DNS服务器 |
| **142.250.185.46** | Google的其中一个网站服务器 |

### IP地址的两种类型

**1. 私有IP地址（Private IP）**
- 用在家里、公司里
- 只有局部网络能用
- 例子：192.168.x.x，10.x.x.x
- 你家的WiFi里的电脑都有这种地址

**2. 公网IP地址（Public IP）**
- 在互联网上可以使用
- 全世界唯一
- 例子：你的家庭网络向外的地址
- 你家的路由器有一个公网IP

## 📖 第2部分：你的电脑怎样获得IP地址？

### 过程很简单

1️⃣ **你连接WiFi**
```
你的电脑说："嘿，路由器！我要加入这个网络！"
```

2️⃣ **路由器回答**
```
路由器说："好的！我给你分配一个地址：192.168.1.100"
```

3️⃣ **你的电脑获得IP地址**
```
你的电脑现在有地址了！其他电脑可以和你说话了！
```

### 这个过程叫什么？

叫 **DHCP** (Dynamic Host Configuration Protocol)
- 翻译过来：动态主机配置协议
- 简单说：自动分配IP地址的过程
- 你不用自己设置，路由器自动做

## 📖 第3部分：DNS - 网址翻译机

### 问题：为什么需要DNS？

想象这个情景：

```
❌ 难的方式：
你想看Google，但你要记住：
Google首页：142.250.185.46
Google邮箱：142.250.185.47
YouTube：172.217.27.206
...很多个地址...

你怎么可能全部记住？！
```

```
✅ 简单的方式：
你只需要记得：google.com
把"google.com"输入浏览器
浏览器问DNS："google.com是哪个IP地址？"
DNS回答："是142.250.185.46"
浏览器连接到那个地址
看网站！
```

**DNS就是解决这个问题的！**

### 什么是DNS？

**DNS** = Domain Name System（域名系统）

简单说：**DNS是网址翻译机**

- 输入：网址（google.com）
- 处理：查询
- 输出：IP地址（142.250.185.46）

### DNS怎样工作？

```
你的电脑
   │
   │ 我要访问 google.com
   ↓
你的路由器（本地DNS缓存）
   │
   │ 有缓存吗？有！就是 142.250.185.46
   │ 或者：没有缓存，我去问别人
   ↓
互联网上的DNS服务器
   │
   │ google.com？我知道！是这个IP：142.250.185.46
   ↓
答案回来了！
   │
   ↓
你的浏览器
   │
   │ 现在我知道地址了，我去连接 142.250.185.46
   ↓
Google网站
   │
   ↓
你看到Google首页了！
```

### 常见的DNS服务器

| DNS服务器 | 地址 | 是谁的 |
|---------|------|-------|
| **Google Public DNS** | 8.8.8.8 | Google的 |
| **Cloudflare DNS** | 1.1.1.1 | Cloudflare的 |
| **OpenDNS** | 208.67.222.222 | OpenDNS的 |

你可以选择用哪个DNS服务器。很多人用Google的（8.8.8.8），因为快而且可靠。

## 📖 第4部分：一个完整的例子

### 你在做什么：输入网址

```
你的浏览器
   │
   │ 你输入：www.baidu.com
   │
   ↓
浏览器的工作流程：
```

### 步骤1：DNS查询

```
浏览器问：www.baidu.com的IP地址是什么？

         你的电脑
           │
           ↓
       路由器/DNS
       (可能有缓存)
           │
          没有缓存
           ↓
      互联网DNS服务器
           │
           ↓
       查数据库...
           ↓
       找到了！220.181.38.148
           │
           ↓
       答案返回给你
```

### 步骤2：建立连接

```
浏览器现在知道：www.baidu.com = 220.181.38.148

浏览器连接到 220.181.38.148
```

### 步骤3：发送请求

```
浏览器发送：
"嘿，220.181.38.148！请给我www.baidu.com的首页"
```

### 步骤4：收到网页

```
服务器回答：
"好的！这是HTML代码、图片、CSS样式..."

浏览器接收所有数据，显示出网页
```

### 步骤5：看到网页

```
你看到百度首页了！✨
```

### 整个过程只需要 **几百毫秒**（0.3秒左右）！

## 📝 关键概念总结

| 概念 | 简单解释 |
|-----|--------|
| **IP地址** | 网络上的地址，像邮编一样 |
| **IPv4** | 现在最常用的IP格式（4段数字） |
| **私有IP** | 只在局部网络用，如192.168.1.x |
| **公网IP** | 在互联网上唯一的地址 |
| **DHCP** | 自动分配IP地址 |
| **DNS** | 网址翻译机，把网址翻成IP地址 |
| **域名** | 网址，如google.com |
| **DNS服务器** | 存储和翻译网址的大电脑 |

## 🎯 本周任务

### 任务1：查看你的IP地址（5分钟）

**Windows用户：**
1. 打开命令提示符（按 Win+R，输入cmd，回车）
2. 输入：`ipconfig`
3. 找到"IPv4 地址"，就是你的私有IP

**Mac用户：**
1. 打开终端
2. 输入：`ifconfig | grep inet`
3. 找到 inet 后面的数字

**记下来：** 我的IP地址是 ___________________

### 任务2：查看你的公网IP地址（3分钟）

1. 打开谷歌
2. 搜索："my ip"
3. 看第一个结果

**记下来：** 我的公网IP地址是 ___________________

### 任务3：做DNS查询实验（10分钟）

**Windows用户：**
1. 打开命令提示符
2. 输入：`nslookup google.com`
3. 看看Google.com对应的IP地址

**Mac/Linux用户：**
1. 打开终端
2. 输入：`dig google.com`
3. 看看结果

**记下来：** google.com的IP地址是 ___________________
